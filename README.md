# Job-Resume Shortlisting MVP

Two-stage candidate matching pipeline.

1. **Stage 1 ‚Äì retrieval**: dense embeddings (`BAAI/bge-large-en-v1.5`, 1024 dims) hosted in Elasticsearch with hybrid scoring (kNN cosine + BM25 over `resume_text`, `resume_skills_norm`, `resume_titles_norm`).
2. **Stage 2 ‚Äì ranking**: LightGBM `LGBMRanker` (LambdaRank / NDCG) reranks the retrieved pool and exposes per-feature SHAP contributions.

## Repository Layout

```
src/
  ml/             # LightGBM training, feature engineering, evaluation
  embeddings/     # BGE encoding + Elasticsearch index builder
backend/
  app/            # FastAPI service (auth, jobs, shortlist, cabinet, stats)

```

## Data

- `data/Clear/jobs_clean.csv` ‚Äî 2 296 unique vacancies (`job_id`, `job_title`, `job_description`, `job_text`, `job_skills_norm`, `job_years_required`).
- `data/Clear/resumes_clean.csv` ‚Äî 18 174 unique resumes (`person_id` ‚Üí `resume_id`, `skills` (phrase bag), `past_titles`, `total_years_experience`, `resume_text`).
- `data/pair_features_labeled.csv` ‚Äî 114 800 labeled (job, resume) pairs with ranking features and 3-class `final_label` (0/1/2). Exactly 50 candidates per job; 2 267 / 2 296 jobs have all three classes. All 8 791 labeled resume IDs are present in the resume pool.

## Training Pipeline

The ranker reads `data/pair_features_labeled.csv` directly ‚Äî no extra ETL is needed.

```powershell
# 1. ML toolkit (PyTorch + sentence-transformers + LightGBM)
python -m pip install -r requirements-ml.txt

# 2. Encode jobs and resumes with BGE-large (auto-detects CUDA)
python -m src.embeddings.build

# 3. Index resumes into Elasticsearch (hybrid: dense_vector + BM25 text)
python -m src.embeddings.index_elasticsearch --recreate-index

# 4. Train LightGBM LambdaRank reranker (20-trial random search by default)
python -m src.ml.train
```

Tunables:

```powershell
python -m src.ml.train --n-trials 30 --n-estimators 3000 --early-stopping 150
python -m src.embeddings.build --batch-size 16 --max-seq-length 256
```

Training outputs (regenerated each run):

- `models/lgbm_ranker.joblib` ‚Äî trained `LGBMRanker`
- `models/ranker_features.joblib` ‚Äî feature column order
- `models/ranker_metadata.json` ‚Äî best params, valid/test metrics, feature importance, all trial records
- `models/metrics/shap_global_summary.csv` ‚Äî global SHAP per feature (consumed by `/stats/explanations/global`)
- `models/metrics/per_job_metrics.csv` ‚Äî per-query NDCG/Precision/Recall/MAP/MRR on test split
- `models/metrics/validation_predictions.csv`, `models/metrics/summary.json`

## Runtime Flow

`job_id` ‚Üí load job + embedding ‚Üí hybrid Elasticsearch retrieval (kNN + BM25) ‚Üí rebuild ranking features ‚Üí engineered per-query features ‚Üí LightGBM scoring + SHAP ‚Üí fusion with retrieval signal ‚Üí top-K shortlist JSON.

The API only performs inference; it does not retrain, re-embed, or re-index during requests.

## Feature Set

19 ranker features split into base + engineered (`src/ml/config.py`):

| Group | Features |
| --- | --- |
| Base | `embedding_cosine`, `skill_overlap_count`, `skill_overlap_ratio`, `title_overlap_ratio`, `years_gap`, `experience_match_flag`, `resume_years_experience`, `job_years_required`, `retrieval_rank` |
| Engineered | `embedding_cosine_norm`, `log_retrieval_rank`, `abs_years_gap`, `skill_overlap_x_emb`, `experience_x_skill`, `title_x_emb`, `embedding_cosine_zscore_in_job`, `skill_overlap_ratio_zscore_in_job`, `title_overlap_ratio_zscore_in_job`, `embedding_cosine_rank_in_job` |

LambdaRank `label_gain = [0, 1, 5]` ‚Äî strongly emphasizes the top label class (2) over partial matches (1).
